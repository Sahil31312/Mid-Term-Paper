1 · Development Method
LLM-Augmented Semantic Knowledge Graph (LASKG) — a 5-stage pipeline alternative to PrimeKG's manual ETL
Stage 1
Ontology-First Schema
Define nodes & edges using formal OWL ontologies before data ingestion
Stage 2
SPARQL Harvesting
Federated queries to distributed endpoints (UniProt, Wikidata, Bio2RDF)
Stage 3
LLM Literature Mining
BioGPT / PubMedBERT extract novel associations from PubMed abstracts
Stage 4
Dual-Format Storage
Neo4j for Cypher queries + RDF/Turtle for OWL reasoning
Stage 5
CI/CD Validation
Monthly re-harvest, GNN link prediction, curator review
